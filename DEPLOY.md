# 🚀 SwiftShip — Deployment Guide (Render.com)

## Prerequisites
- GitHub account with this project pushed to a repository
- Render.com account (free tier works)
- Node.js 18+ and Composer installed locally

---

## 🗂️ Step 1 — Prepare the Repository

```bash
# Make sure these files exist at project root
ls render.yaml Procfile .env.example DEPLOY.md

# Install dependencies locally to verify no errors
composer install
npm install && npm run build

# Commit everything (never commit .env)
git add .
git commit -m "feat: production deployment config"
git push origin main
```

---

## 🌐 Step 2 — Create Project on Render

1. Go to [render.com](https://render.com) and sign in
2. Click **"New +"** → **"Blueprint"**
3. Connect your GitHub account if not already done
4. Select your repository
5. Render will auto-detect `render.yaml` and show services to create:
   - `swiftship-app` (Web Service)
   - `swiftship-db` (PostgreSQL Database)
6. Click **"Apply"** — Render provisions both automatically

---

## ⚙️ Step 3 — Environment Variables

After services are created, go to Web Service → **Environment** tab and verify:

| Key | Value |
|---|---|
| `APP_ENV` | `production` |
| `APP_DEBUG` | `false` |
| `APP_KEY` | *(auto-generated by render.yaml)* |
| `APP_URL` | *(your Render URL)* |
| `DB_CONNECTION` | `pgsql` |
| `DB_SSLMODE` | `require` |
| `CACHE_DRIVER` | `database` |
| `SESSION_DRIVER` | `database` |
| `SESSION_COOKIE` | `swiftship_session` |
| `QUEUE_CONNECTION` | `database` |
| `LOG_CHANNEL` | `stderr` |

> ⚠️ If `APP_KEY` is empty, run locally:
> ```bash
> php artisan key:generate --show
> ```
> Copy output and paste into Render environment variables.

---

## 🔄 Step 4 — First Deploy

1. Go to Web Service → **"Manual Deploy"** → **"Deploy latest commit"**
2. The build runs:
```
composer install
npm install && npm run build
php artisan config:cache
php artisan route:cache
php artisan view:cache
php artisan migrate --force
php artisan db:seed --force
```

---

## ✅ Step 5 — Verify

Once build shows **"Live"**:

1. Visit Render URL → SwiftShip homepage
2. Test public tracking: `/track/IND250512ABCDEF`
3. Test admin login: `/login`
   - Default admin: `admin@logistics.test` / `password`

## 🛠️ Troubleshooting

### 500 error
```bash
# Most common: APP_KEY missing
php artisan key:generate --show
# Paste into Render env vars, redeploy
```

### Database connection error
- Verify `DB_SSLMODE=require` is set
- Check DB host/port/name match Render PostgreSQL connection details

### Assets 404
```bash
# Verify vite.config.js has outDir: 'public/build'
# Verify layouts use @vite() helper
# Redeploy — npm run build must complete successfully
```

### Migrations failed
```bash
php artisan migrate:status
php artisan migrate --force
```

### Free tier sleep (15 min inactivity)
- Upgrade to Starter ($7/mo) for always-on
- Or use [UptimeRobot](https://uptimerobot.com) free ping every 10 mins

## 📦 Local Development

```bash
git clone https://github.com/yourusername/swiftship.git
cd swiftship
composer install
cp .env.example .env
# Edit .env for local MongoDB
php artisan key:generate
php artisan db:seed
npm install && npm run dev
php artisan serve
# Visit http://localhost:8000
```
